Utiliser Git et ses commandes

L'arbre Git

Git permet de gérer des versions différentes(branches) d'un projet. C'est utile pour éviter les erreurrs ou pour gérer le travail de son équipe. On fusionne les branches au Master quand on est satisfait du résultat. En effet, on peut imaginer Git comme un arbre. Il est composé d'un tronc et de branches. Le tronc (master) représente le projet tel qu'il est sur internet. Les branches elles, sont des versions du site avec des modifications ou du contenu en plus. Elles attendent validation avant d'être mélangée(merged) au master. On peut créer une branche par fonctionnalité, ou par membre de l'équipe .. A vous de voir comment vous voulez vous organiser.

Mettre en place un projet avec Git

Sur GitHub il faut créer un répertoire pour votre projet. Cela va générer une url qui correspond à votre site. Ensuite il faut téléverser tous les fichiers déjà existant de votre site. Maintenant, depuis votre ordinateur allez dans votre terminal de commande(tapez cmd dans vos recherches sur windows). Sans rien faire d'autre tapez "git clone urlgithub". Votre projet est prêt ! Il vous suffit maintenant de vous déplacer jusqu'à l'emplacement de votre projet pour travailler dessus.

Les commandes Git

Voici une liste des commandes les plus utiles d'après moi pour maintenir votre projet à jour.

  • git branch nomdelabranche - Créer une nouvelle branche
  • git checkout nomdelabranche - Déplace vers la branche
  • git pull - Récupère la version du projet de GitHub
  • git status - Vérifie si il y a eu des modifications
  • git add . - Sélectionne tous les fichiers qui ont été modifié
  • git commit -m"j'ai modifié ceci" - Prépare les fichiers pour la publication
  • git push - Met les modifications en ligne
